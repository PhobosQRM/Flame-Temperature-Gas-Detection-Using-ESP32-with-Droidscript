IOT-BASED ENVIRONMENTAL HAZARD DETECTION SYSTEM USING ESP32 AND DROIDSCRIPT

Project Overview

The IoT Fire Monitoring System is a comprehensive safety solution designed to detect early signs of environmental hazards. By leveraging the power of the ESP32 microcontroller and LittleFS technology, this system provides real-time surveillance of temperature, smoke, and open flames.

Unlike traditional passive alarms, this system creates a responsive dashboard hosted directly on the device (Localhost), allowing users to monitor conditions via their mobile phones while simultaneously providing immediate physical alerts through a custom LED indicator system and a buzzer.

Block Diagram Description

Overview

The system architecture centers on the ESP-WROOM-32 microcontroller, which acts as the central processing unit. The data flow operates in three distinct stages:

Input Stage: Three environmental sensors continuously feed data into the system:

  • MQ2 Gas Sensor: Calibrated to detect Methane gas particles and smoke.
  • DHT11 Sensor: Continuously measures ambient Temperature and Humidity.
  • Flame Sensor: Detects infrared wavelengths (760nm - 1100nm) to identify open fire sources.

Processing Stage:

  • The ESP32 reads these analog and digital inputs, processing the logic to determine if safety thresholds have been breached. It also hosts the web server.

Output Stage: The system provides dual-layer feedback:

  • Digital (Web): A Localhost Website communicates via Wi-Fi to display real-time data on a mobile interface.
  • Physical (Alerts): LED Indicators and a Buzzer connected to digital output pins provide immediate visual and audible feedback based on specific sensor values.

Figure 2: System Circuit Diagram

Circuit Diagram Description

Overview

The hardware implementation is built around the ESP-WROOM-32 Dev Module. The circuit focuses on modularity and immediate visual identification of faults.

Sensor Integration:

  • The Flame Sensor and MQ2 Gas Sensor are connected to the ESP32's analog/digital pins to monitor hazardous elements.
  • The DHT11 Sensor is wired for digital signal transmission to report climate data.

Visual Status System:

  • Safety Indicator: A central Green LED remains active to confirm the system is powered and the environment is safe.
  • Fault Indicators: The circuit features Three Individual Red LEDs, each paired with a specific sensor (Flame, Gas, and Temp). If a sensor detects a danger, its corresponding Red LED activates, allowing the user to instantly pinpoint the source of the problem.

Power Logic:

All components share a common Ground (GND) and are powered via the ESP32's 3.3V and 5V voltage rails.
